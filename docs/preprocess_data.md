# Data Preprocessing

This document describes how LightDFN converts dataset manifests into packed HDF5 files for training.

## Purpose

The preprocessing stage reads the manifest files generated by `scripts/datasets/download_datasets.sh`, validates every referenced audio file, resamples audio when required, and writes three training-ready HDF5 containers:

- `speech_clean.hdf5`
- `noise_music.hdf5`
- `rir.hdf5`

These files are consumed directly by the training pipeline and are intended to reduce repeated decoding and improve I/O throughput.

## Command

Run preprocessing with the repository script and explicit CLI flags:

```bash
scripts/datasets/build_hdf5.sh \
  --data-dir "./datasets/" \
  --num-workers 4 \
  --dtype float32 \
  --profile prototype
```

Example with explicit overrides:

```bash
scripts/datasets/build_hdf5.sh \
  --data-dir "./datasets/" \
  --profile production \
  --num-workers 8 \
  --sr 48000 \
  --dtype float32 \
  --max-freq -1 \
  --force-copy-cfg
```

## Processing Flow

### 1. Resolve Paths And Profile

`scripts/datasets/build_hdf5.sh` derives the manifest, HDF5, and configuration paths from `--data-dir` unless explicit overrides are provided.

- `--profile prototype` uses the lightweight training configuration.
- `--profile production` uses the full training configuration.
- `--profile apple` lowers worker count and limits the maximum frequency to reduce memory pressure.

### 2. Copy The Dataset Configuration

The script copies `datasets/configs/<profile>/dataset.cfg` to the configured output path when the destination file does not already exist. Use `--force-copy-cfg` to overwrite it intentionally.

### 3. Validate Manifests

The build script expects these manifest files:

- `lists/clean_all.txt`
- `lists/noise_music.txt`
- `lists/rir_all.txt`

Each manifest is validated before packing begins.

### 4. Build The HDF5 Datasets

For each manifest, `scripts/datasets/prepare_data.py`:

- resolves and validates every input path;
- loads audio with `torchaudio` and `soundfile`;
- resamples to the target sample rate when needed;
- encodes audio into the requested storage format; and
- writes the result into the target HDF5 container.

The build creates one output per data category:

- clean speech from `clean_all.txt` -> `hdf5/speech_clean.hdf5`
- noise and music from `noise_music.txt` -> `hdf5/noise_music.hdf5`
- room impulse responses from `rir_all.txt` -> `hdf5/rir.hdf5`

## Optional Flags

The main options exposed by `scripts/datasets/build_hdf5.sh` are:

- `--data-dir`: base directory containing `lists/` and receiving `hdf5/`
- `--profile`: `prototype`, `production`, or `apple`
- `--sr`: target sample rate
- `--dtype`: output storage type, such as `int16` or `float32`
- `--num-workers`: number of preprocessing workers
- `--max-freq`: upper frequency bound stored in HDF5 metadata
- `--subsample`: keep every Nth item from each manifest for quick smoke-test builds
- `--force-copy-cfg`: replace the existing output config with the selected profile template

## Notes

- Use `--subsample` for a quick validation run before building the full corpus.
- Keep manifest paths stable after download; preprocessing expects the listed audio files to remain available.
- Review `scripts/datasets/build_hdf5.sh --help` for the complete flag set.
